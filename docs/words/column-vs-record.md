# カラム値 vs レコード

- minesql では「カラム値」と「レコード」を明確に区別している
- カラム値 (`[][]byte`): アクセスメソッドおよびそれよりも上位のレイヤーで扱われる概念。各カラムの値をフラットな配列で保持する
- レコード (`node.Record`): アクセスメソッドよりも下位のレイヤー (B+Tree) で扱われる概念
  - B+Tree のレコードを示す (Header/Key/NonKey の 3 領域で構成される)(詳細は [B+Tree のレコード](../architecture/storage/btree/node/record.md) を参照)
- アクセスメソッドは両者の境界に位置し、上位層から受け取ったカラム値を B+Tree レコードにエンコードし、B+Tree レコードをカラム値にデコードして上位層に返す
