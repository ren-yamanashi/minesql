# 統計情報取得メソッドの配置先

## Motivation

コストベース最適化に必要な統計情報 (リーフページ数、B+Tree の高さ) を取得するメソッドをどのレイヤーに配置するか決める必要があった

## Decisions

Access Method レイヤー (`TableAccessMethod`, `UniqueIndexAccessMethod`) にメソッドを配置する

## Context

統計情報の取得には B+Tree のメタページへのアクセスが必要になる。このメソッドの配置先として以下の 2 案を検討した

### 案 1: Access Method レイヤーに配置

`TableAccessMethod.LeafPageCount()` や `UniqueIndexAccessMethod.Height()` として実装する

### 案 2: Catalog レイヤーに配置

`TableMetadata` にメソッドを追加し、内部で Access レイヤーに委譲する

### 判断結果

- Access Method レイヤーの責務: Access Method レイヤーは B+Tree 操作のラッパーとして `Insert/Delete/Update/Search/Create` を既に提供しており、`LeafPageCount` や `Height` も同じ性質の操作である
- Catalog レイヤーの責務: Catalog はテーブル名・カラム定義・インデックス定義といった静的なスキーマ情報を管理しており、実行時の物理状態 (ページ数やツリーの高さ) は責務の範囲外である

責務的に Access Method レイヤーに実装するのが適切であると判断したため、案 1 を採用することにした
