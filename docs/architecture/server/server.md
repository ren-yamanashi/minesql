# サーバー

## 概要

minesql はクライアント・サーバーモデルで動作する。サーバーは TCP 上で独自のシンプルなプロトコルを用いてクライアントと通信し、受け取った SQL を解析・実行して結果を返す。

### コネクション管理

- クライアントからの TCP 接続を受け付け、接続ごとに独立した goroutine で処理する
- 各接続にはセッションが紐づき、トランザクション状態を保持する
- 一定時間 (60 秒) 操作がない場合、接続をタイムアウトで切断する
- 切断時にアクティブなトランザクションがあれば自動でロールバックする

## プロトコル

クライアントとサーバー間の通信には TCP 上の独自プロトコルを使用する。パケットは固定長ヘッダー + 可変長ボディの構造を持つ。

| フィールド | サイズ | 内容 |
| --- | --- | --- |
| Header | 4 bytes | ボディの長さ (Big Endian) |
| Body | N bytes | ペイロード (SQL またはレスポンス) |

### リクエスト

- Header: Body のデータ長 (4 バイト、Big Endian)
- Body: SQL クエリ文字列

### レスポンス

- Header: Body のデータ長 (4 バイト、Big Endian)
- Body: 実行結果の文字列 (CSV 形式)
  - エラー時は `Error: <メッセージ>` 形式

## シャットダウン

- `Ctrl+C` などの OS シグナルを受信した際に、サーバーはグレースフルシャットダウンを行う
- 具体的には、バッファプール内のダーティページをすべてディスクにフラッシュしてからプロセスを終了する
