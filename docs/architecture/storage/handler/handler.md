# Handler

## 概要

- ストレージ層のリソースをシングルトンで管理する
- サーバー起動時にストレージ層のリソースを初期化し、クエリ間で共有する状態を保持する
- テーブル操作やインデックス操作においては、クエリ間で情報を共有する必要がないため、アクセスメソッド (`access`) が直接提供し、エグゼキュータやプランナーから参照される
- 一方で、トランザクション制御や統計情報の管理、DDL 操作などはクエリ間で共有する必要があるため、Handler が提供する

## シャットダウン

正常終了時、Handler は以下の順序で終了処理を行う

1. バッファプール内の全ダーティーページをディスクにフラッシュする
2. 全ディスクを Sync する
3. REDO ログをクリアする

REDO ログがクリアされることで、次回起動時にクラッシュリカバリが不要であることを判定できる
