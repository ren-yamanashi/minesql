# レコード挿入

## 全体の流れ

1. ルートノードから再帰的にブランチノードを辿り、挿入先のリーフノードに到達する
2. リーフノードにレコードを挿入する
   - リーフノードが満杯の場合は分割が発生し、境界キーが親ブランチノードに伝播する
3. 親ブランチノードに境界キーが伝播された場合、ブランチノードにレコードを挿入する
   - ブランチノードも満杯であれば分割が発生し、さらに上の親に伝播する (再帰的)
4. ルートノードまで分割が伝播した場合、新しいルートノード (ブランチノード) を作成する

## リーフノードへの挿入

- リーフノードが満杯でない場合
  - Slotted Page の仕様に従って、リーフノードのボディ部分にレコードを挿入する

- リーフノードが満杯の場合
  - 新しいリーフノードを作成し、自身のリーフノードのレコードのうち、半分を新しいリーフノードに移動させる
  - 新しいリーフノードに、挿入対象のレコードを挿入する

### 分割挿入の流れ

#### 1. 分割挿入先のリーフノードを作成する

- 新しいノードは古いノードの左側に配置される (古いノードより小さい値が格納される)

#### 2. 古いノードのレコードを新しいノードに移動しつつ、挿入対象のレコードを適切な位置に挿入する

- 挿入対象のレコードのキーと、分割前のリーフノード (古いノード) の先頭のキーを比較する
  - 古いノードの先頭レコードのキー < 挿入対象のキー: 古いノードの先頭レコードを分割先のリーフノード (新しいノード) に移動する
    - その過程で、新しいノードが半分埋まったら、挿入対象のレコードを古いノードに挿入して、ループ終了
  - 古いノードの先頭レコードのキー >= 挿入対象のキー: 挿入対象のレコードを新しいノードに挿入する
    - その後、新しいノードが半分埋まるまで、古いノードの先頭レコードを新しいノードに移動する

#### 3. 新しいリーフノードの先頭のキー値を親ノードに伝える

- リーフノードの分割が生じた際は、ブランチノードが持つべき境界値 (key) を増やす必要がある
- つまりブランチノードのレコードを増やす必要がある
- この時のブランチノードのレコードの key (=境界値) は、新しく作られたリーフノードの先頭のキーの値になる
- そのため、新しく作られたリーフノードの先頭のキー値を親ノード (ブランチノード) に伝える

**_以下例_**

_例1: 挿入対象のキーが中間にある場合_

```txt
初期状態:
  - 古いノード (満杯): [10, 20, 30, 40, 50]  ← newRecord = 25 を挿入したい
  - 新しいノード (空):  []

ループ 1回目:
  - 新しいノードが半分埋まっているか? → No (空なので)
  - 古いノードの先頭(10) < 挿入対象(25) → 古いノードの先頭レコード(10)を新しいノードに移動
  - 結果
    - 古いノード: [20, 30, 40, 50]
    - 新しいノード: [10]

ループ 2回目:
  - 新しいノードが半分埋まっているか? → No (まだ半分未満)
  - 古いノードの先頭(20) < 挿入対象(25) → 古いノードの先頭レコード(20)を新しいノードに移動
  - 結果
    - 古いノード: [30, 40, 50]
    - 新しいノード: [10, 20]

ループ 3回目:
  - 新しいノードが半分埋まっているか? → No (まだ半分未満)
  - 古いノードの先頭(30) > 挿入対象(25) → 挿入対象(25)を新しいノードに挿入
  - その後、新しいノードが半分埋まるまで古いノードの先頭レコードを移動する
    - 新しいノード [10, 20, 25] → 半分埋まったので移動終了
  - 結果
    - 古いノード: [30, 40, 50]
    - 新しいノード: [10, 20, 25]
```

_例2: 挿入対象のキーが先頭より小さい場合_

```txt
初期状態:
  - 古いノード (満杯): [10, 20, 30, 40, 50]  ← newRecord = 5 を挿入したい
  - 新しいノード (空):  []

ループ 1回目:
  - 新しいノードが半分埋まっているか? → No (空なので)
  - 古いノードの先頭(10) >= 挿入対象(5) → 挿入対象(5)を新しいノードに挿入
  - その後、新しいノードが半分埋まるまで古いノードの先頭レコードを移動する
    - 1回目の移動: 古いノードの先頭(10)を新しいノードに移動
      - 古いノード: [20, 30, 40, 50], 新しいノード: [5, 10]
    - 2回目の移動: 古いノードの先頭(20)を新しいノードに移動
      - 古いノード: [30, 40, 50], 新しいノード: [5, 10, 20]
    - 新しいノードが半分埋まったので、移動終了
  - 結果
    - 古いノード: [30, 40, 50]
    - 新しいノード: [5, 10, 20]
```

## ブランチノードへの挿入

- ブランチノードが満杯でない場合
  - Slotted Page の仕様に従って、ブランチノードのボディ部分にレコードを挿入する

- ブランチノードが満杯の場合
  - 新しいブランチノードを作成し、分割挿入を行う

### 分割挿入の流れ

#### 1. 分割挿入先のブランチノードを作成する

- 新しいノードは古いノードの左側に配置される (古いノードより小さい値が格納される)

#### 2. 古いノードのレコードを新しいノードに移動しつつ、挿入対象のレコードを適切な位置に挿入する

- 挿入対象のレコードのキーと、古いノードの先頭のキーを比較する
  - `古いノードの先頭レコードのキー < 挿入対象のキー` の場合
    1. 古いノードの先頭レコードを新しいノードに移動する
    2. この移動を繰り返し、新しいノードが十分に埋まったら (後述の境界キー取り出しが可能になったら)、挿入対象のレコードを古いノードに挿入して、ループ終了
  - `古いノードの先頭レコードのキー >= 挿入対象のキー` の場合
    1. 挿入対象のレコードを新しいノードに挿入する
    2. その後、新しいノードが十分に埋まるまで、古いノードの先頭レコードを新しいノードに移動する

#### 3. 新しいノードの末尾レコードを境界キーとして取り出す

- 新しいノードの末尾レコードを取り出し、そのキーを親ノードに伝播させる境界キーとする
- 取り出したレコードの非キー (子ノードの PageId) を、新しいブランチノードの RightChildPageId として設定する

**_例: 挿入対象のキーが中間にある場合_**

```md
初期状態:
  - 古いノード (満杯): [(10,P1), (20,P2), (30,P3), (40,P4), (50,P5)]  ← newRecord = (25,P6) を挿入したい
  - 古いノードの RightChildPageId = P0
  - 新しいノード (空): []

レコードの移動と挿入 (上述のステップ 2 に該当)
  - 古いノードの先頭(10) < 挿入対象(25) → (10,P1) を新しいノードに移動
  - 古いノードの先頭(20) < 挿入対象(25) → (20,P2) を新しいノードに移動
  - 古いノードの先頭(30) >= 挿入対象(25) → (25,P6) を新しいノードに挿入
  - 新しいノードが十分に埋まるまで古いノードの先頭レコードを移動
  - 結果:
    - 古いノード: [(40,P4), (50,P5)], RightChildPageId = P0
    - 新しいノード: [(10,P1), (20,P2), (25,P6), (30,P3)]

境界キーの取り出し (上述のステップ 3 に該当)
  - 新しいノードの末尾レコード (30,P3) を取り出す
  - 境界キー = 30 (親ノードに伝播する)
  - P3 は新しいノードの RightChildPageId として設定する
  - 結果:
    - 古いノード: [(40,P4), (50,P5)], RightChildPageId = P0
    - 新しいノード: [(10,P1), (20,P2), (25,P6)], RightChildPageId = P3
    - 親に伝播する境界キー: 30
```

### リーフノードの分割挿入との違い

- リーフノードの分割挿入では「新しいノードが半分埋まったか」でループを終了する
- ブランチノードの分割挿入では、ループ終了後に末尾レコードを境界キーとして取り出す必要があるため「取り出した後も半分以上を維持できるか」でループを終了する
- また、リーフノードの分割挿入の戻り値は「新しいノードの先頭キー」だが、ブランチノードの分割挿入の戻り値は「取り出した末尾レコードのキー (=境界キー)」である

### 再起的なノード分割

- 分割挿入によりブランチノードが増えた場合、その親ノード (ブランチノード) にも新しく境界値となるレコードを挿入する必要がある
- その際に親ノードの容量によっては親ノードの分割挿入も起きる可能性がある

### ルートノードの分割

- ルートノードが分割された場合、新しいルートノード (ブランチノード) を作成する
- 新しいルートノードは以下を持つ
  - 分割によって返されたキー
  - 古いルートノードの PageId (左の子)
  - 新しく作成されたノードの PageId (右の子)
- また、メタページのルートページ ID を新しいルートノードの PageId に更新する必要がある
