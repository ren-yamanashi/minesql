# レコード

## 概要

- B+Tree のノードには、複数のレコードが格納される
- レコードはヘッダー・キーフィールド・非キーフィールドの 3 領域で構成される
  - B+Tree はヘッダーサイズとキーフィールドサイズを保持し、ヘッダーをスキップしてキーフィールド部分でソート順を決定する
  - B+Tree はヘッダーやフィールドの中身の意味は関知しない
- キーフィールドは Memcomparable format でエンコードされる ([参照](../../../../about/memcomparable-format.md))
  - 複合キーの場合も正しくソートされる

## シリアライズ形式

レコードは以下の形式でバイト列にシリアライズされ、[Slotted Page](./slotted-page.md) に格納される

| フィールド | バイト数 | 説明 |
| --- | --- | --- |
| header | 可変 | ヘッダー領域。B+Tree はヘッダーサイズ分をスキップしてキー比較を行う |
| key fields | 可変 | キーフィールドを Memcomparable format でエンコードして連結したもの |
| non-key fields | 可変 | 非キーフィールドを連結したもの |

## ブランチノード

- キーフィールド: 子ノードを分割する境界値
- 非キーフィールド: key 未満の値を持つ子ノードのページ ID

※詳細は [ブランチノード](./branch-node.md) を参照

## リーフノード

- レコードにどのようなフィールドを格納するかは、B+Tree を利用する側 ([アクセスメソッド](../../../access.md)) が決める
  - B+Tree はリーフノードのレコード内のフィールドの意味については関与しない
