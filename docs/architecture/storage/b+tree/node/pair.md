# key-value ペア

## 参考文献

- [The physical structure of records in InnoDB](https://blog.jcole.us/2013/01/10/the-physical-structure-of-records-in-innodb/)

## 概要

- B+Tree のノードには、複数の key-value ペアが格納される
- key-value のペアはそれぞれエンコードされる
  - キーが複合キーの場合も考慮して (複合キーの場合も正しくソートされるように) Memcomparable format を使用してキーをエンコードしている ([参照](../about/memcomparable-format.md))

### ヘッダー

| フィールド | 説明 |
| --- | --- |
| `delete_mark` | 削除マーク。削除された行はこのフラグが立つ。物理削除は後でパージによって行われる |

### クラスタ化インデックス (リーフノード)

- key: テーブルのプライマリキーの値をエンコードしたものが格納される
- value: 以下の内容をエンコードしたものが格納さらえる

| フィールド | バイト数 | 説明 |
| --- | --- | --- |
| `last_modified` | 8 | 自分を最後に変更したトランザクション ID |
| `roll_ptr` | 8 | 自分が更新される 1 世代前の行の内容 (undo ログ) へのポインタ (= ロールバックポインタ) |
| 非キーカラム値 | 可変 | プライマリキー以外のカラム値を結合したものをエンコードしたもの |

### セカンダリインデックス (リーフノード)

- key: セカンダリインデックスを構成するカラム値をエンコードしたものが格納される
- value: プライマリキーをエンコードしたものが格納される (`last_modified` や `roll_ptr` はセカンダリインデックスのペアには格納されない)
