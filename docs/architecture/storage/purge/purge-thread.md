# パージスレッド

## 概要

- 大きく分けて以下の 2 つの処理を行う
  - UNDO ログのメンテナンス
  - 削除マーク (`deleteMark`) がついたレコードの処理

### UNDO ログのメンテナンス

- UNDO ログは行やインデックスの更新が発生する度に増加するため、放置されるとファイルサイズが肥大化し、ストレージを浪費する
- そのため、不要になった UNDO ログを定期的に削除する必要がある
- 以下のすべての条件を満たした時、UNDO ログは不要になる
  - コミットされ、ロールバックが発生しなくなった時
  - 他のすべてのトランザクションがその過去を参照しなくなった時
    - ※各レコードは自分が更新される一世代前のレコードの内容 (UNDO ログ) へのポインタを持っているため、それが参照されている間は UNDO ログを削除できない
